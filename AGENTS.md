## Проект: Alien Grover

Браузерная игра-тамагочи в Sci-Fi/Alien стилистике (Vanilla HTML/CSS/JS, без сборки, без зависимостей).

### 🛸 Тематика и СТРОГИЙ КОНТЕНТНЫЙ ЗАПРЕТ
- Это **безопасный reskin** для публикации в Telegram: пользователь выращивает **фантастическое инопланетное растение** (Звездоцвет, Ксено-Корень, Спора) в био-камере космической станции.
- **КАТЕГОРИЧЕСКИ ЗАПРЕЩЕНО** использовать любые реальные наркотические термины: `weed`, `каннабис`, `марихуана`, `гров`, `гровер`, `гров-бокс`, `куст`, `шишки`, `тГК`, ` Skunk`, `OG Kush`, `Gorilla Glue`, `White Widow`, `AK-47`, `Northern Light` и т.п.
- Запрещены эмодзи и иконки, ассоциируемые с каннабисом: `🍁`, `🌿`, `💨`, `🪴` (как символ выращивания), `🌾`. Допустимо `🌾` только в нейтральном контексте, не связанном с культурой.
- Разрешённые Sci-Fi термины и образы:
  - Организм: **культура, ксено-организм, биомасса, спора, Звездоцвет**.
  - Ресурсы: **гидро-плазма** (бывшая вода), **био-катализатор** (бывшая еда), **свет** (LED), **HP**.
  - Продукт: **ксено-экстракт, звёздная пыль, споры** (вместо «шишки»).
  - Действия: **Залить** (плазму), **Подкорм** (катализатором), **Извлечь** (споры/экстракт).
  - Локация: **био-камера** выращивания (вместо «гров-бокс»), **лаборатория** (магазин), **борт-журнал** (статистика).
  - Игрок: **исследователь / пилот** (вместо «гровер»).
- При любом редактировании текстов/переводов/замен изображений — прогоняй `grep` по запрещённым словам перед сдачей.

### Визуальный стиль · НЕОНОВЫЙ КОСМОС (Alien Sci-Fi)
- Палитра: чёрно-космический фон `#03030a` со звёздной россыпью, **неоновые акценты** — циан `#00e5ff`, зелёный `#39ff14`, пурпур `#a855f7`, маджента `#ff00e5`, золото `#ffd000`, алый `#ff3b6b`.
- Текст светится неоновыми `text-shadow` (cyan/purple glow). Бордюры тёмные (`#0a0a14`), chunky-кнопки сохраняют solid-shadow механику.
- LED-фитолампа в био-камере: неоновый glow `0 0 22px #a855f7` + radial purple→green — единственное место активного неона (как в оригинале).
- Куст культуры: тот же SVG-веер из 7 листочков, но стебель `linear-gradient(#a855f7 → #39ff14)` с лёгким purple glow; капсула-поддон `#3a3c44→#1f2127` со встроенным cyan-подсветом inset.
- Стадии `plant--s0..s3` и body-language `plant--happy/thirsty/hot/droopy` — в CSS, не трогать без необходимости.
- Шрифт: `Impact, 'Arial Black', 'Haettenschweiler', system-ui` weight 900, uppercase — для кнопок/заголовков/HUD/HP-бара.

### Файлы
- `index.html` — разметка (HUD, селектор камер, культура, шкалы, модалки intro/lab/log, toasts)
- `style.css` — неоновый космический chunky-стиль внутри телефонного фрейма `max-width:400px`
- `game.js` — вся логика: game loop, состояние, сохранения, события, звуки (WebAudio-synth)

### Запуск и тестирование
- Сервер: `python3 -m http.server 8765` из корня репо, URL `http://localhost:8765/`
- `file://` в Playwright **заблокирован** — только через локальный HTTP-сервер
- Проверка UI: Playwright navigate → snapshot; для инспекции состояния использовать `evaluate` (модель не читает скриншоты)
- Состояние хранится в `localStorage` под ключом `alienGroverSave`. Для чистого теста: `localStorage.clear()` через evaluate → `location.reload()`
- Для теста TWA вне Telegram: в Playwright использовать `page.addInitScript` **до** `goto` (см. детальный блок в archives оригинального проекта), затем `renderHUD()` для обновления имени.

### Telegram Web App (TWA)
- Скрипт `https://telegram.org/js/telegram-web-app.js` подключён в `<head>` `index.html`. Вне Telegram этот URL недоступен — это нормально (404/timed out в console не блокирует игру).
- В `game.js`: `const tg = window.Telegram?.WebApp ?? null` и `const tgUser = tg?.initDataUnsafe?.user ?? null` — frozen на загрузке. При наличии вызываются `tg.ready()`, `tg.expand()`, `tg.disableVerticalSwipes()` (все в `try/catch`).
- Имя игрока: `tgUser.first_name` выводится в `#hudUser` (stat 👽 с подписью «Пилот»). Вне Telegram `#hudUser` получает `aria-hidden="true"` и скрывается. Установка через `renderHUD()` — НЕ выносить в отдельную функцию, иначе нарушится диф-рендер паттерн.
- `body` зафискирован: `overflow:hidden` + `touch-action:none`. Скролл разрешён ТОЛЬКО у `.pots__list` (`touch-action:pan-x`) и `.modal__card` (`touch-action:auto`).

### Архитектура (важные неочевидные факты)
- `state` объявлен `let` в `game.js` и дополнительно экспортирован как `window.state` для отладки в evaluate.
- Game loop: `setInterval(loop, 1000)` тикает шкалы; `setInterval(save, 10000)` автосохранение; `beforeunload` экстренный сейв.
- **Диф-рендер:** `render()` не пересоздаёт DOM. `renderPots()` строит 5 `<li>` один раз (`potsUIInit`), дальше diff. `renderPlant()` кэширует `lastStage/lastBodyClass/lastSpeech/lastMood/lastStrainId/lastLightOn` и меняет `className` только при реальной смене → CSS keyframes (`leafSway`, body-language) не прерываются. `setMeter()`/`renderHUD()`/`renderProgress()` проверяют `textContent`/width перед записью. `renderActionHints()` через `classList.toggle`. НЕ переписывать на полный rebuild — это убьёт плавность анимаций.
- В `game.js` есть отладочные хелперы `window.dev`: `money(n)`, `level(n)`, `kill()`, `event()`, `clearSave()` — использовать в Playwright evaluate для быстрого тестирования.
- Культура нарисована инлайн-SVG в `index.html` (7 листочков, веер), стиль/анимации в `.plant__leaves` и `.plant__svg` в `style.css`. **Стадии `plant--s0..s3` и body-language `plant--happy/thirsty/hot/droopy` — в CSS, не трогать без необходимости.**
- Сорта, апгрейды, камеры, достижения определены как константы-массивы в начале `game.js` (`STRAINS`, `UPGRADES`, `POTS`, `ACHIEVEMENTS`); баланс — там же в `CONFIG.rates`. Сорта названы в Sci-Fi стиле: Звездоцвет-Альфа, Аврора-Нексус, Крио-Спора, Пульсар-47, Орион-Гало, Квазар-Голиаф.
- Цикл роста ~5 минут (`CONFIG.rates.growth: 0.0024`); пост-извлекающая модалка `#harvestModal` с 3 кнопками (🧬 Запустить/🛒 В лабораторию/✕), автозапуска НЕТ — `doHarvest()` обнуляет камеру и зовёт `openHarvestModal()`.
- `ensureFirstPlant()` в BOOT: если камера 0 свободна и Owned — авто-сажает `star1` (для старта/мёртвого сейва).
- Мягкий старт: первые 30 минут decay ×0.5 (`starterDecay: true` у Звездоцвет-Альфа, проверка `p.ageSec < 1800`).

### Соглашения по стилю
- В `game.js` используется `const $ = (s, el=document) => el.querySelector(s)` и `$$` — локальные селекторы, не путать с jQuery.
- Сообщения пользователю — через функцию `toast(text, kind)`, kinds: `""`/`"good"`/`"level"`/`"bad"`/`"ach"`.
- Звук — `SFX.<name>()` (WebAudio-синтез, без ассетов), отключается `state.muted`.
- Подтверждения (`confirm`) допустимы для сброса прогресса и замены культуры.

### Границы изменений
- Геймплей/баланс → править `game.js` (данные в начале + `tickPlant`, `loop`).
- Визуал культуры → `index.html` (SVG-блок `.plant__leaves`) + `style.css` (`.plant__*`).
- UI/модалки/toasts → `index.html` + `style.css`; открытия/закрытия модалок — функции `open*`/`close*` в `game.js`.
- При изменении `index.html`/`style.css` — **сохранять все `id`, `data-meter`, классы стадий/body-language, классы `btn--glow/pulse/lamp-on`, `shop__item--locked/owned`, `event--luck`, `toast--*`, `dot--*`, `gb__*`** — на них завязан JS. Перед сдачей проверять через Playwright `evaluate` что селекторы живы.
- **Любые новые тексты проверять `grep`-ом по списку запрещённых наркотических терминов** (см. блок «СТРОГИЙ КОНТЕНТНЫЙ ЗАПРЕТ» выше) — сдать результат можно, только если ни одно запрещённое слово не встречается.
