## Проект: Hot Pepper

Браузерная игра-тамагочи: симулятор выращивания экстремально острых перцев чили (Vanilla HTML/CSS/JS, без сборки, без зависимостей).

### 🌶️ Тематика
- Браузерная игра-тамагочи: игрок-фермер выращивает острые перцы чили в горшках/мини-теплицах.
- **Валюта:** `state.money` (старт 50). HUD `#hudMoney` рисует 🌶️ без `$` (эмодзи в `.stat__icon`, значение в `.stat__value`). Тосты/магазин тоже используют 🌶️.
- **Прогресс:** шкала «Жгучесть · SHU» в UI (`index.html`), но в коде сортов НЕТ поля `shu` — награда за сбор = `Math.round(strain.baseReward * (p.health/100))`.
- **Ресурсы/кнопки (UI):** Полить (💧), Удобрить (🧪), Свет (☀️), Собрать (🔥).
- **Сорта:** в `STRAINS` 6 реальных сортов чили по нарастанию остроты — Халапеньо (`star1`), Серрано (`aurora`), Кайен (`cryo`), Хабанеро (`pulsar`), Призрачный перец (`orion`), Каролина Рипер (`quasar`). ID остались от alien-схемы (`star1..quasar`) — НЕ переименовывать, завязаны на сейвы. Поля `baseReward`/`greenZone`/`minLevel`/опц. `starterDecay`.
- **Локации/игрок:** горшок (UI «Горшки»), лаборатория (магазин, 🛒), журнал (📊), фермер (🧑‍🌾).
- **Эмодзи-словарь (огненный/лабораторный):** 🌶️, 🔥, ☀️, 💧, 🧪, 📊, 🛒, 🥵, 🥀, 💀, 🏆, ⭐, 💯, 🛡️.
- **Reskin (Alien Grover → Hot Pepper) ЗАВЕРШЁН:** визуал (`index.html`/`style.css`/SVG перец), HUD и `game.js` полностью Hot Pepper. `storageKey` переименован в `"hotPepperSave"` с авто-миграцией со старого `"alienGroverSave"` в `load()` (читает старый → пишет новый → удаляет старый). Все alien-имена/иконки/тосты заменены. Запрещённых эмодзи в коде нет. При любых новых текстах всё равно сверять с `grep`.
- **Запрещённые эмодзи (целевой запрет):** 🛸, 👽, 🧬, 💠, ✦, 🌌, 🌟, 💡 (как фитолампа), 🪴 (как «гров-бокс»). В текущем коде отсутствуют — не вводить.
- **Наркотических терминов в проекте НЕТ**, но при любом вводе текстов прогоняй `grep` по списку: `weed`, `каннабис`, `марихуана`, `гров`, `гровер`, `куст`, `шишки`, `тГК`, `Skunk`, `OG Kush`, `Gorilla Glue`, `White Widow`, `AK-47`, `Northern Light`. Сдать можно, только если ни одно слово не встречается.

### Визуальный стиль · PREMIUM MINIMALIST (Hot Pepper)
- Палитра: глубокий чёрно-серый фон `#0a0a0c` / панели `#141416` / карточки `#1c1c20`; **неоновые акценты** — алый `#ff3b30`, огненно-оранжевый `#ff8c00`, янтарь `#ffb300`. Текст `#f5f5f7`, dim `#8e8e93`.
- Стиль строгий, минималистичный: тонкие границы `1px solid rgba(255,255,255,.08)`, мягкие тени `0 4px 20px rgba(0,0,0,.45)`, скругления `14–20px`. Никаких «chunky» solid-shadows.
- Активные акценты светятся мягким `box-shadow: 0 0 16px rgba(255,59,48,.45)` и `text-shadow: 0 0 10px rgba(255,140,0,.5)` — применяем сдержанно (кнопка света в on, harvest pulse, danger-зона шкал).
- LED-фитолампа в горшке: в выключенном состоянии — тёмная `#1c1c20`; во включённом — неоновый glow `0 0 22px rgba(255,140,0,.85)` + radial orange→amber — единственное место активного неона.
- Перец — инлайн-SVG в `index.html` (плод с изгибом + плодоножка-коронка + блик + пульсирующий glow), обёрнут в `<g class="leaves">`. Градиент тела `#43e36b → #ffb300 → #ff8c00 → #ff5a1f → #ff3b30` (зелёный → янтарь → алый). Стебель `.plant__stem` `linear-gradient(#43e36b, #ff3b30)`. Микроанимации: `leafSway` (покачивание `.leaves`) + `pepperPulse` (пульсация `.pepper-pulse`).
- Стадии `plant--s0..s3` и body-language `plant--happy/thirsty/hot/droopy` — в CSS, не трогать без необходимости.
- Шрифт: системный sans `-apple-system, Inter, system-ui`; для цифр/HUD/заголовков — `font-variant-numeric: tabular-nums`. Uppercase Минь только для bead/labels, размер шрифта компактный (11–18px).
- **Плавные анимации изменения чисел:** все числовые показатели (капсаицин, XP, уровень, SHU, процентов шкал) твинятся `requestAnimationFrame` через helper `tweenNum(el, to, formatter)` (ease-out-cubic, ~400ms). Не перезаписывать значение напрямую без твина.

### Файлы
- `index.html` — разметка (HUD, селектор горшков, перец, шкалы, модалки intro/lab/log, toasts)
- `style.css` — premium minimalist тёмный стиль внутри телефонного фрейма `max-width:400px`
- `game.js` — вся логика: game loop, состояние, сохранения, события, звуки (WebAudio-synth), твины чисел

### Запуск и тестирование
- Сервер: `python3 -m http.server 8765` из корня репо, URL `http://localhost:8765/`
- `file://` в Playwright **заблокирован** — только через локальный HTTP-сервер
- Проверка UI: Playwright navigate → snapshot; для инспекции состояния использовать `evaluate` (модель не читает скриншоты)
- Состояние хранится в `localStorage` под ключом `CONFIG.storageKey` (сейчас `"hotPepperSave"`). При загрузке автоматически мигрируется со старого ключа `"alienGroverSave"` (читается → копируется в новый → старый удаляется). Для чистого теста: `localStorage.clear()` через evaluate → `location.reload()` (в Playwright очищать через `page.addInitScript` ДО `goto`, иначе `beforeunload` старой страницы успевает сохранить).
- Для теста TWA вне Telegram: в Playwright использовать `page.addInitScript` **до** `goto`, затем `renderHUD()` для обновления имени.

### CI / Deploy
- `.github/workflows/static.yml`: на каждый push в `main` — артефакт из корня репо деплоится в GitHub Pages (без build-шага, Vanilla JS уезжает как есть). Включить Pages: Settings → Pages → Source = GitHub Actions.
- Секрет `TELEGRAM_BOT_TOKEN` заведён в GitHub Secrets репо (для будущего backend), в текущем workflow не используется.

### Telegram Web App (TWA)
- Скрипт `https://telegram.org/js/telegram-web-app.js` подключён в `<head>` `index.html`. Вне Telegram этот URL недоступен — это нормально (404/timed out в console не блокирует игру).
- В `game.js`: `const tg = window.Telegram?.WebApp ?? null` и `const tgUser = tg?.initDataUnsafe?.user ?? null` — frozen на загрузке. При наличии вызываются `tg.ready()`, `tg.expand()`, `tg.disableVerticalSwipes()` (все в `try/catch`).
- Имя игрока: `tgUser.first_name` выводится в `#hudUser` (stat 🧑‍🌾 с подписью «Фермер»). Вне Telegram `#hudUser` получает `aria-hidden="true"` и скрывается. Установка через `renderHUD()` — НЕ выносить в отдельную функцию, иначе нарушится диф-рендер паттерн.
- `body` зафискирован: `overflow:hidden` + `touch-action:none`. Скролл разрешён ТОЛЬКО у `.pots__list` (`touch-action:pan-x`) и `.modal__card` (`touch-action:auto`).

### Архитектура (важные неочевидные факты)
- `state` объявлен `let` в `game.js` и дополнительно экспортирован как `window.state` для отладки в evaluate.
- Game loop: `setInterval(loop, 1000)` тикает шкалы; `setInterval(save, 10000)` автосохранение; `beforeunload` экстренный сейв.
- **Диф-рендер:** `render()` не пересоздаёт DOM. `renderPots()` строит 5 `<li>` один раз (`potsUIInit`), дальше diff. `renderPlant()` кэширует `lastStage/lastBodyClass/lastSpeech/lastMood/lastStrainId/lastLightOn` и меняет `className` только при реальной смене → CSS keyframes (`leafSway`, body-language) не прерываются. `setMeter()`/`renderHUD()`/`renderProgress()` используют `tweenNum()` для плавных чисел + сравнение перед запуском. `renderActionHints()` через `classList.toggle`. НЕ переписывать на полный rebuild — это убьёт плавность анимаций.
- В `game.js` есть отладочные хелперы `window.dev`: `money(n)`, `level(n)`, `kill()`, `event()`, `clearSave()` — использовать в Playwright evaluate для быстрого тестирования.
- Перец — инлайн-SVG в `index.html` (внутри `.plant__leaves`, группа `<g class="leaves">`, эллипс `.pepper-pulse`), стиль/анимации в `.plant__svg` / `.plant__svg .leaves` / `.pepper-pulse` в `style.css`. **Стадии `plant--s0..s3` и body-language `plant--happy/thirsty/hot/droopy` — в CSS, не трогать без необходимости.**
- Сорта, апгрейды, горшки, достижения определены как константы-массивы в начале `game.js` (`STRAINS`, `UPGRADES`, `POTS`, `ACHIEVEMENTS`); баланс — там же в `CONFIG.rates`. У сорта поля `baseReward`/`greenZone`/`minLevel`/опц. `starterDecay` (поля `shu` НЕТ).
- Цикл роста ~30 минут (`CONFIG.rates.growth: 0.056`); `growthBad: 0.0093`. Пост-сборочная модалка `#harvestModal` с 3 кнопками (🌶️ Посадить снова/🛒 В лабораторию/✕), автозапуска НЕТ — `doHarvest()` обнуляет горшок и зовёт `openHarvestModal()`.
- `ensureFirstPlant()` в BOOT: если горшок 0 свободен и Owned — авто-сажает `star1` (для старта/мёртвого сейва).
- Мягкий старт: первые 30 минут decay ×0.5 (`starterDecay: true` у стартового сорта `star1`, проверка `p.ageSec < 1800`).
- **Достижение survivor** отслеживается через `const _survivors = new Set()` (в памяти, рядом с `hasOwned`) — индексы горшков, где растение упало ниже 12 HP. При восстановлении >30 HP → `state.stats.saves++` + удаление из Set. НЕ сохраняется в localStorage (раньше протекало через `p._saveTagged` — вычищено). Очищается при смерти/сборе.

### Соглашения по стилю
- В `game.js` используется `const $ = (s, el=document) => el.querySelector(s)` и `$$` — локальные селекторы, не путать с jQuery.
- Сообщения пользователю — через `toast(text, kind)`, kinds: `""`/`"good"`/`"bad"`/`"ach"`.
- Звук — `SFX.<name>()` (WebAudio-синтез, без ассетов), отключается `state.muted`.
- Подтверждения (`confirm`) допустимы для сброса прогресса и замены перца.

### Границы изменений
- Геймплей/баланс → править `game.js` (данные в начале + `tickPlant`, `loop`).
- Визуал перца → `index.html` (SVG внутри `.plant__leaves`, группа `<g class="leaves">`, эллипс `.pepper-pulse`) + `style.css` (`.plant__*`, `.pepper-pulse`).
- UI/модалки/toasts → `index.html` + `style.css`; открытия/закрытия модалок — функции `open*`/`close*` в `game.js`.
- При изменении `index.html`/`style.css` — **сохранять все `id`, `data-meter`, классы стадий/body-language, классы `btn--glow/pulse/lamp-on`, `shop__item--locked/owned`, `event--luck`, `toast--*`, `dot--*`, `gb__*`** — на них завязан JS. Перед сдачей проверять через Playwright `evaluate` что селекторы живы.
- **Любые новые тексты проверять `grep`-ом по списку запрещённых наркотических терминов** (см. блок «Тематика» выше) — сдать результат можно, только если ни одно запрещённое слово не встречается.
